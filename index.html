Mail Merge & Dynamic Content

Create multiple documents simultaneously.

Mail Merge can address a variety of usage scenarios within an enterprise - from automated form letter generation to the printing of product catalogs and everything in between. As you would expect, our mail merge engine allows you to bind the WinForms Rich Text Editor to any external data source and preview content generated via merge operations within a given document template. Integrated mail merge tools include:

  • Mail Merge Ribbon UI
  • Preview Merged Data
  • Merge to a New Document

You can also add fields (dynamic elements) to your document. Fields are placeholders that store and display data from a data source. The DevExpress WinForms Rich Text Editor automatically updates fields when the document is saved, printed, or when a mail merge operation is executed.

Protection & Security

Protect your work. Prevent unauthorized access.

Password-protect your documents and prevent unauthorized use. If you apply document protection on the RTF control's Review tab, you can mark a document as read only (you can optionally allow users to add comments to a protected document).

In addition, document protection allows you to restrict editing to specific portions of a given document (you can control what an end-user can and cannot edit).

You can also encrypt your documents with a password and/or configure the actions available to users. Unavailable operations will be hidden from corresponding context menus and disabled within the control’s Ribbon UI.
